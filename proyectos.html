Laberinto programado en C++

Para comenzar la sección de proyectos en este proyecto hablare sobre como tuve que aplicar lógica, algoritos y sobre todo sudor y coraje para hacerlo. Este proyecto se hizo para la materia de Estructura de Datos, durante el proyecto se me pidio realizar un laberinto aleatorio con el uso de pilas u colas en el lenguaje de programación C++. Para este proyecto hubo una serie de complicaciones bastante graves debido a que debia estudiar varios algoritmos de lógica para que simplemente el laberinto funcionara normalmente y de forma correcta como por ejemplo el algoritmo de backtracking, esto claro que fue una tarea bastante complicada y tardada porque primero debia aprender como funcionaba el algoritmo y luego aplicarlo en el código que era algo simplemente dificl. Además de que el lenguaje de programación donde se hizo toda la tarea no era para nada beneficioso para mi debido a que no tenía casí experiencia con C++ como si tengo algo de experiencia en Java y Python, tambíen que era de los lenguajes de programación que mas odio... por malas experiencias en preparatoria como mi primer lenguaje de programación. Durante la realización del proyecto se me pidio que el laberinto fuera aleatorio y que este tenga con algunas restricciones. Como el que hay un total de 3 puertas en todo el laberinto una de colores direntes, Rojo, Azul y Verde. Estas puertas deberian ser abiertas con llaves del mismo color que tambien estan colcadas de forma random en el laberinto decido para el proyecto tambien utilizar una gran cantidad de mensajes de depuración para indicar a quien este viendo el código qué celda ha sido recorrida o se regreso y en caso de que haya una llave tomarla y seguir a abrir la puerta. El proyecto fue muy dificil para mi debido a que yo me encargue de casí todo lo más dificil como lo fue en su totalidad el código.

Control Vehicular

Para este proyecto se nos pidio realizar un sistema en que sirviera para poder servir como Control Vehicular, este proyecto fue requerido para la materia de Sistemas en Internet se nos pidio una serie de requerimientos como poder realizar consultas de propietarios de autos, consultas del auto, sacar licencias de conducir así como tambien algunas otras tareas comunes en estos lugares. Se nos pidio que este sistema sea completeamente funcional y que tambien administremos su base de datos. Para la realización de este proyecto hice uso del lenguaje de consultas SQL, con el servidor local XAMMP y sus herramientas Apache y PHPmyAdmin, además de que se deberian hacer labores de programación BACKEND y FRONTEND, durante el desarrollo del proyecto sin duda lo más complicado fue todo el BACKEND y la base de datos debido que nuevamente el encargado de todo esto fui yo, no sabía nada de backend y sobre todo el lenguaje de programación que nos fue impuesto por el profesor fue PHP. Un lenguaje de programación bastante comodo si lo puedo deicr ya que si puede ser algo dificl al Inicio pero termine usando PHP para multiples proyectos que necesitara la comunicación con PHPmyAdmin es así que durante la ejecución del proyecto se nos hizo incapie en tambien proporcionar un manúal de usuario funcional y sobre todo detallado del sistema y como funcionara es así que esto tambien... lo hice yo con la esperanza de ser lo más claro con este proyecto. Es así que las herramientas complicadas de usar fueron en su totalidad las que estaban relacionadas con el BACKEND como PHP y SQL, debido a que no sabiamos mucho de este debido a algunos problemas con nuestras clases de base de datos.
